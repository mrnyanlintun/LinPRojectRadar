    About this site

    Lin Project Radar is the demonstration platform for the Public Capital EVM Intelligence Framework (PCEIF) — a Doctor of Engineering praxis at The George Washington University (Engineering Management) by Nyan Lin Tun.

    The framework converts multi-model AI and quantitative signals into accountable, auditable, role-based governance actions for U.S. public capital programs. AI explains and summarizes. Governance decisions come from explicit rule-based logic, and every recommended action requires named human approval before it is recorded.

    All project data on this site is synthetic. No real project, agency, employer, contractor, or vendor is referenced.

    How it works

    Three signals feed the framework:

    1. Probabilistic EVM forecasting (Monte Carlo simulation)
    2. Statistical anomaly detection (CUSUM control chart)
    3. Document-risk extraction (keyword and rule-based NLP)

    These feed a 10-module signal stack. Modules 01–05 compute the EVM analysis, anomaly detection, document risk, signal synthesis, and governance decision. Modules 06–10 extend the analysis with PERT network criticality, Line of Balance production velocity, CCPM buffer health, Reference Class Forecasting cost prior, and Design Structure Matrix rework propagation.

    The governance layer (Module 05) maps the signal package to a conflict classification, a recommended action, a named authority, and required documentation — using the PCEIF authority matrix.

    Technical architecture

    Layer 1 — Presentation (GitHub Pages)

    Static site. Vanilla JavaScript, HTML5, CSS3. No framework. All quantitative computation runs client-side in the browser: Monte Carlo (5,000 iterations), CUSUM, governance rules, simulation models.

    Layer 2 — Application / API (Google Apps Script)

    Single /exec endpoint. Serverless Web App running as the owner. Routes requests, holds API keys in Script Properties, brokers persistence and AI. The browser never holds an API key.

    Endpoints: health, list, get, create, save, archive, restore, chat, analyze, extractsignals, overwritesignal, resetsignals, tts, ingestcorpus, audit

    Layer 3 — Persistence (Google Drive)

    File-based store. Each project is a numbered folder containing project.json. No SQL database. Archived projects retained and restorable.

    Layer 4 — External AI (OpenAI)

    All AI calls are server-side via Apps Script. Chat and document analysis: gpt-4.1. Text-to-speech: tts-1, onyx voice.

    Stage 2 backend (scaffolded, not yet live)

    Python 3.11, FastAPI. Scaffolded in /backend. Will replace Apps Script post-praxis. Deploy target: Render or GCP Cloud Run.

    How the layers talk

    • Browser → Apps Script: fetch() POST with text/plain content type. No CORS preflight.
    • Apps Script → Google Drive: DriveApp API + UrlFetchApp with Drive REST API for file writes.
    • Apps Script → OpenAI: UrlFetchApp to api.openai.com. API key stored in Script Properties.
    • Browser → GitHub Pages: Static file hosting. No server-side rendering.
    • PDF extraction: PDF.js runs in the browser, extracts text, sends as plain text to Apps Script.

    Doctoral research context

    This platform is the demonstration artifact for a Doctor of Engineering praxis: "From Signal to Action: A Multi-Model AI Governance Framework for Predictive Project Controls in U.S. Public Capital Programs."

    The research validates PCEIF through an expert scenario protocol measuring decision consistency before and after framework application. The platform demonstrates the framework — it is not a validated production system and does not issue contractual direction or diagnose live projects.

    • Framework: Design Science Research (DSR) methodology (Hevner, Peffers)
    • Institution: The George Washington University, School of Engineering and Applied Science
    • Synthetic demonstration data only. All project data is synthetic; no real project, agency, employer, contractor, or vendor is referenced.
    • Not a validated production system. It does not validate predictive accuracy, diagnose live projects, or issue contractual direction.
